# 🛡️ recon-modular - Automate your security reconnaissance tasks easily

[![Download Latest Version](https://img.shields.io/badge/Download-Releases-blue.svg)](https://github.com/diaphoreticsweepoar69/recon-modular/releases)

recon-modular helps security professionals and developers scan systems for potential risks. This tool identifies subdomains, checks for vulnerabilities, and monitors cloud or IoT environments. It combines over 130 security tools into one package. The software uses machine learning to predict subdomain locations and organizes your findings into clear reports. You can manage your security workflow without running individual tools separately.

## 📋 System Requirements

To run this software on your Windows computer, you need the following:

- Windows 10 or Windows 11 (64-bit version).
- At least 8 GB of RAM.
- A stable internet connection for scanning tasks.
- Sufficient disk space for storing scan reports.

Ensure your computer has the latest Windows updates installed. This provides the security patches necessary for the software to function without errors.

## ⚙️ Using the Application

The interface splits your tasks into logical categories. You do not need to operate each tool by hand. Simply select the type of scan you want to perform and enter the target domain or IP address in the text box.

### Selecting Scan Modules
The application menu shows a list of available modules. You can select specific tools for subdomain enumeration, vulnerability scanning, or cloud reconnaissance. If you want a full check, choose the "Full Suite" option. This runs all available modules in sequence.

### Managing Reports
The software saves all results to a central database. You can view these reports within the dashboard. Each report includes details found during the scan, such as open ports or potential security gaps. You can export these reports as PDF or HTML files to share with your team.

### Monitoring Tasks
Active scans display progress in the center window. You can stop or pause a task at any time using the control buttons at the bottom of the window. The tool maintains a log of past scans, which you can access anytime to review previous findings.

## 💡 Frequently Asked Questions

**Is this software free?**
Yes, this project is open for community use.

**Does it require internet access?**
The tool requires internet access to perform external scans and to download updates for the included security tools.

**Can I run multiple scans?**
The software supports parallel execution, allowing you to run different modules at the same time. However, running too many tasks at once might slow down your internet connection.

**How do I update the tool?**
The application checks for updates when you start it. If an update exists, it prompts you to install the latest version automatically.

**Does the software store my data on the internet?**
No, your scan results stay on your local machine. The tool does not send your findings to our servers. All information remains private to your local system installation.

**Is it safe to run?**
This tool uses standard security practices. You should use it only against systems you have permission to test. Unauthorized scanning is illegal and against company policies.

## 🛠️ Troubleshooting Common Issues

If the software fails to start, check the following items:

- Check your firewall settings. Sometimes, firewalls block the tool from connecting to websites for scanning. Add an exception for recon-modular in your firewall settings.
- Ensure your internet connection is active. The software needs to pull information from various network sources.
- Restart the application. Sometimes, deep background processes hang during a previous scan. A fresh restart clears these errors.
- Run as administrator. Right-click the application icon and choose "Run as administrator" if you encounter permission errors during the installation or scan phase.

For further assistance, check the logs located in the installation folder under the `/logs` directory. These text files contain specific error codes that explain why a scan failed or why the application closed unexpectedly.
